# Android Mobile Agent 使用说明

## 一、用途
手机端执行移动触达任务，电脑后台只负责创建任务。

执行边界：
- 自动：拉任务、复制话术、打开 TikTok/Zalo/WhatsApp
- 人工：点击发送，然后在 App 标记“已发送”

## 二、准备
1. 后台数据库已执行：
   - `php database/run_migration_mobile_outreach.php`
2. 后台设备表 `mobile_devices` 已有记录：
   - `agent_token`
   - `device_code`
3. 手机安装最新 APK。

## 三、手机端配置
打开 App，填写：
- 后台地址：`https://your-domain.com` 或 `https://your-domain.com/admin.php`
- 设备 Token：`mobile_devices.agent_token`
- 设备编码：`mobile_devices.device_code`
- 轮询间隔：建议 `2-5` 秒
- 任务类型：按需勾选

点“保存配置”，再点“启动 Agent”。

## 四、执行流程
1. 后台创建移动任务（达人页或外联工作台）。
2. 手机自动拉到任务并：
   - 复制文案到剪贴板
   - 打开目标渠道
3. 人工在目标 App 完成发送。
4. 回到 Agent，点“标记已发送”。

可选动作：
- “重新打开目标”
- “标记跳过”
- “标记失败”

## 五、常见问题
### 1) 任务创建了但手机没执行
- 检查 Token/Device Code 是否匹配后台设备记录
- 检查手机网络可访问后台地址
- 检查 App 状态是否显示 `Running`

### 2) 出现 HTML 返回而不是 JSON
- 通常是后台地址写错（应指向 `admin.php`）
- 或设备 Token 无效

### 3) 不能自动发送？
- 当前版本设计就是“自动填充 + 人工发送”，不做无人值守自动发送
