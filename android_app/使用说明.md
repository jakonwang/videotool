# Android 应用使用说明

## 项目概述

这是一个简单的 WebView 套壳 Android 应用，用于显示视频管理系统的平台分类和视频列表，避免浏览器广告干扰。

## 功能特点

✅ **平台分类展示** - 从服务器获取平台列表，以卡片形式展示  
✅ **视频播放** - 点击平台后，使用 WebView 加载视频页面  
✅ **无广告干扰** - 使用 WebView 而非浏览器，避免广告弹窗  
✅ **返回导航** - 支持返回按钮和系统返回键  
✅ **下载支持** - 支持视频和封面下载功能  

## 使用步骤

### 1. 配置服务器地址

打开 `app/src/main/java/com/videotool/MainActivity.java`，找到第 20 行：

```java
private static final String BASE_URL = "https://your-domain.com";
```

修改为你的实际服务器地址，例如：
```java
private static final String BASE_URL = "https://videotool.banono-us.com";
```

### 2. 编译 APK

#### 使用 Android Studio（推荐）

1. 下载并安装 [Android Studio](https://developer.android.com/studio)
2. 打开项目文件夹 `android_app`
3. 等待 Gradle 同步完成
4. 点击 `Build` -> `Build Bundle(s) / APK(s)` -> `Build APK(s)`
5. APK 文件位置：`app/build/outputs/apk/debug/app-debug.apk`

#### 使用命令行

```bash
cd android_app
./gradlew assembleDebug
```

APK 文件位置：`app/build/outputs/apk/debug/app-debug.apk`

### 3. 安装到手机

1. 将 APK 文件传输到 Android 手机
2. 在手机上启用"未知来源"安装
3. 点击 APK 文件进行安装

## 应用界面

### 主界面（平台列表）
- 显示所有可用的平台
- 以 2 列网格布局展示
- 每个平台显示为卡片样式

### 视频界面
- 使用 WebView 加载视频页面
- 支持视频播放、下载等功能
- 点击返回按钮可返回平台列表

## API 接口说明

应用需要以下 API 接口：

### 1. 获取平台列表
- **URL**: `/api/platforms`
- **方法**: GET
- **返回格式**:
```json
{
  "code": 0,
  "data": [
    {
      "id": 1,
      "name": "TikTok",
      "code": "tiktok",
      "icon": "🎵"
    }
  ]
}
```

### 2. 视频页面
- **URL**: `/?platform={平台代码}`
- **方法**: GET
- **说明**: 返回视频播放页面（HTML）

## 注意事项

1. **服务器地址配置**：确保修改 `BASE_URL` 为正确的服务器地址
2. **HTTPS 支持**：建议使用 HTTPS，如果使用 HTTP，需要在 AndroidManifest.xml 中配置网络安全策略
3. **网络权限**：应用已自动添加网络权限
4. **最低版本**：支持 Android 5.0 (API 21) 及以上版本

## 常见问题

### Q: 应用无法连接服务器？
A: 检查服务器地址是否正确，确保服务器可以正常访问。

### Q: 平台列表为空？
A: 检查 API 接口 `/api/platforms` 是否正常返回数据。

### Q: 视频页面无法加载？
A: 确保服务器支持 HTTPS，或配置网络安全策略允许 HTTP。

### Q: 如何修改应用名称和图标？
A: 
- 修改应用名称：编辑 `app/src/main/res/values/strings.xml`
- 修改应用图标：替换 `app/src/main/res/mipmap/ic_launcher.png`
