# 饰品款式（耳环、手链、项链等）编号表与 CSV 导入

若使用 Excel（如本地的款式表），可在 Excel 中 **另存为 CSV（UTF-8）**，再导入后台。**勿将超大 xlsx 提交到 Git**（仓库已忽略 `docs/*.xlsx`）。

**重要：CSV / 纯文本 `.txt` 不能保存「插入到单元格里的图片」**，另存为 CSV 后图片列会变成空或丢失。若款式图是 **嵌入在 Excel 格里** 的，请 **直接上传 `.xlsx` / `.xls` / `.xlsm`**（不要先另存 CSV）。若坚持用 CSV，请在「图片」列填写 **可访问的 http(s) 链接**、**站内路径** 或 **Base64**。

确保至少包含以下含义的列：

| 列（表头可含下列关键词） | 必填 | 说明 |
|--------------------------|------|------|
| 产品编号 | 是 | 与仓库一致的款号 / ID |
| 图片 | 是 | 支持 `http(s)` 完整链接、站内路径（如 `/uploads/xxx.jpg`）、或 Base64 图片数据列 |
| 爆款类型 | 否 | 展示在搜索结果中 |

将保存的 `.csv` 在后台 **素材 → 寻款** 中上传导入。图片链接需部署环境可访问（本机路径请放在 `public` 下并用 `/` 开头的 Web 路径填写）。

使用 **Excel .xlsx** 时，支持 **浮动插入图** 与 **「将图片放置在单元格中」**（Microsoft 365）；后者由 PhpSpreadsheet 读入为单元格内对象，后台导入已按单元格值解析，无需再改成「浮动」样式。
